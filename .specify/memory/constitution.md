# FEMAT Constitution

## Core Principles

### I. User-First Design
Toutes les fonctionnalités doivent être conçues en pensant d'abord à l'expérience utilisateur. L'interface doit être intuitive, accessible et responsive.

### II. Performance & SEO
Le site doit être rapide, optimisé pour le SEO, et accessible sur tous les appareils. Utilisation de la génération statique (SSG) quand possible.

### III. Maintainability
Code propre, bien structuré, avec des composants réutilisables. Documentation claire des fonctionnalités complexes.

### IV. Accessibility
Respect des standards WCAG pour l'accessibilité web. Support des lecteurs d'écran et navigation au clavier.

### V. Mobile-First
Design responsive avec approche mobile-first. Le site doit être parfaitement utilisable sur mobile.

## Technology Stack

- **Frontend**: Nuxt.js 4.x avec Nuxt UI
- **Language**: TypeScript
- **Styling**: Tailwind CSS (via Nuxt UI)
- **Build**: Vite (via Nuxt)
- **Deployment**: Vercel avec Static Site Generation (SSG)
  - Plateforme: Vercel (https://vercel.com)
  - Documentation: https://nuxt.com/deploy/vercel
  - Configuration: Zero-config, détection automatique de Nuxt.js

## Development Workflow

1. Spécification avant implémentation (Spec-Driven Development)
2. Tests avant déploiement
3. Code review pour les fonctionnalités importantes
4. Documentation des décisions architecturales importantes

## Governance

Cette constitution guide toutes les décisions de développement. Les amendements doivent être documentés et justifiés.
